# 独立游戏开发计划：切片阶段（第 1-6 周）

## 背景

- 单人独立开发，目标平台 Steam PC，买断制
- 每周 10-15 小时，工作日晚上 + 周末
- 有其他游戏引擎经验，Godot 零基础但不需要长时间学习
- 核心玩法和探索切片的设计尚未完成，有想法需要梳理
- 美术风格：手绘/卡通，AI 生成为主 + 关键资产外包
- 引擎：Godot 4 + GDScript

---

## 第一周：选定 agent + 配置记忆 + 切片大方向

**目标：确定长期开发伙伴，让它了解你和项目，定下切片方向**

| 任务 | 工具 | 产出 |
|------|------|------|
| 调研 Codex 费用/能力，和 Claude Code 对比，做决定 | Codex 官网、Claude Code | 选定结果 |
| 把项目背景、设计偏好、已有想法写入 agent 记忆 | Claude Code memory / CLAUDE.md | agent 初始记忆配置 |
| 跟 agent 梳理切片想法，输出大方向文档 | Claude Code | 切片大方向文档 |

---

## 第二周：设计链讨论——定方向

**目标：沿"天津素材→故事→任务→玩法→循环"讨论一遍，每块定方向**

| 任务 | 工具 | 产出 |
|------|------|------|
| 天津素材选材：选哪些地点、文化、美食元素 | Claude Code、网络资料、个人经验 | 素材清单 |
| 故事设计：几条线、什么角色、什么冲突 | Claude Code | 故事框架文档 |
| 任务转化：故事怎么变成玩家可操作的流程 | Claude Code | 任务大纲 |
| 玩法梳理：探索中涉及哪些交互 | Claude Code | 玩法列表 |
| 循环定义：玩家重复执行的核心循环是什么 | Claude Code | 循环描述文档 |

---

## 第三周：展开为可执行设计文档

**目标：把方向细化成可以直接开发的设计稿**

| 任务 | 工具 | 产出 |
|------|------|------|
| 每个任务的具体流程（玩家做什么、系统怎么响应） | Claude Code、Notion/Markdown | 任务流程文档 |
| 每个任务涉及的场景内容清单（地点、物件、NPC） | Claude Code、Excalidraw/纸笔 | 场景内容清单 + 布局草图 |
| 对话大纲（关键对话的内容和分支） | Claude Code | 对话大纲文档 |
| 玩法交互的细节规则（操作方式、判定逻辑、反馈） | Claude Code | 玩法细则文档 |

---

## 第四周：搭环境 + 探索场景灰盒开始

**目标：项目跑起来，做出可行走的灰盒场景**

| 任务 | 工具 | 产出 |
|------|------|------|
| 安装 Godot 4.4+，创建项目，建立文件夹结构，连接 Git | Godot、Git、GitHub | 可运行的空项目 + 远程仓库 |
| 熟悉 Godot 2D 场景搭建、输入处理、角色移动 | Godot 官方文档、Claude Code | 基础操作熟练 |
| 搭建灰盒探索场景：地形、边界碰撞、摄像机跟随 | Godot、Claude Code | 可行走的灰盒场景 |

---

## 第五周：探索场景灰盒——交互与对话

**目标：场景里的东西能交互、NPC 能对话**

| 任务 | 工具 | 产出 |
|------|------|------|
| 实现可交互对象系统：靠近显示提示→按键交互→触发事件 | Godot、Claude Code | 交互系统 |
| 实现 NPC 对话系统：对话数据结构、对话 UI、分支选择 | Godot、Claude Code | 对话系统 |
| 放置测试用 NPC 和可交互物，验证流程 | Godot | 可交互的探索场景 |

---

## 第六周：探索场景灰盒——任务串联

**目标：把设计文档里的任务流程串进场景，跑通完整体验**

| 任务 | 工具 | 产出 |
|------|------|------|
| 实现任务/进度系统：任务状态跟踪、触发条件、完成判定 | Godot、Claude Code | 任务系统 |
| 把设计文档中的任务流程接入场景（NPC 触发→任务推进→结果反馈） | Godot、Claude Code | 完整任务流程可跑通 |
| 自测完整流程，记录问题和需要补的设计 | Godot、Notion/Markdown | 自测报告 + 待补设计清单 |

---

## 管理维护

| 频率 | 任务 | 工具 |
|------|------|------|
| 每次开发结束 | Git commit + push | Git |
| 每周末 | 简短周报（做了什么/卡了什么/下周计划） | Notion 或 Markdown |
| 设计有变更时 | 更新设计文档，不靠记忆 | Claude Code / Notion |
