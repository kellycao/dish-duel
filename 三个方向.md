# 天津菜品游戏 - 核心玩法脑洞记录

## 背景

围绕"以天津这座城市的气质和生活"为主题的游戏，菜品作为切入点。已确认设计原则：

1. **感官要直接打中玩家**——视觉冲击 > 听觉/文字描述
2. **菜本身是主角**——不让饭局规矩、文化仪式抢走焦点
3. **横向广度 > 纵向深度**——琳琅满目，而非一道菜的100种解读
4. **零知识门槛**——好玩在前，懂菜在后
5. **市井为主，但接受魔幻/怪诞包装**

核心玩法只管"好玩"，城市氛围放在**局外获取菜品的探索过程**中解决（B5 一菜一故事作为局外探索层备用）。

下面三个方向作为核心玩法候选，逐个独立推敲。后续将逐个展开讨论。

---

## 方向一 · A4 胃是有限资源

### 为什么玩

"琳琅满目但吃不完"是每个人旅行时的真实焦虑。这个游戏把那种焦虑变成玩法——**FOMO（错过恐惧）本身就是核心驱动力**。玩家不是在完成任务，是在"有限的胃里塞下最满意的一天"。

### 玩法目标

每一局是"一天"（或"一顿"，看颗粒度）。玩家要在**胃容量、时间、金钱、体力**等有限资源下，从可选菜品中规划出"最满意"的食单。满意度不是简单加总——有：
- **搭配加成**：煎饼果子+面茶是经典早点组合，比单吃加分
- **顺序加成**：先汤暖胃再吃重油，满意度高于反过来
- **饱腹惩罚**：越接近胃上限，新菜的满意度越低

### 核心循环

1. **开局**：展示今日可选菜品阵列（视地图区域/时间段/天气/季节而变）
2. **规划**：玩家看菜品的"饱腹值""满意度""搭配关系"，决定吃什么、按什么顺序
3. **执行**：逐个"吃"——每道菜上桌有视觉表现，吃完获得满意度反馈
4. **结算**:日终结算——总满意度、搭配评分、隐藏成就（"一天不吃面"/"连续三顿海河鲜"）
5. **解锁**：高满意度解锁新区域/新菜品/新时段

### 成长空间

- **胃容量成长**：不是简单变大，而是解锁"分餐能力"——把一天分成更多小餐
- **搭配图谱**：玩家逐步解锁菜品之间的搭配关系，从"盲选"走向"精准规划"
- **区域解锁**：一条街 → 一个区 → 全城
- **时间轴解锁**：早餐 → 早+午 → 全天 → 夜宵
- **挑战模式**：限定条件（只吃素/只吃3元以下/只在一条街吃饱）

### 风险与待解决

- 单独撑一个游戏偏薄，"选+吃+算分"循环会疲劳。最适合做**元系统/包装层**
- **"最优解"问题**：搭配关系固定后玩家很快找到最优。需要随机性（今天这家没开/季节限定）或竞争要素（别人先吃了最后一份）

## 方向二 · E16 菜品对决（卡牌）

### 为什么玩

**策略深度 + 收集欲 + "用菜下棋"的新鲜感**。规则做好后，玩家每打一局都在学"什么菜配什么菜更厉害"——这个认知过程本身就是在传达饮食文化，但玩家不觉得自己在学，觉得自己在赢。

**核心定位**：菜不拟人化，菜本身就是牌。表现以菜为主，叙事部分放在局外探索（获取菜品的过程）。

### 玩法目标

用收集到的菜品组成"一桌菜"（牌组），在对局中打赢对手的"一桌菜"。胜负标准是**"谁的这桌菜更让评审满意"**——厨王争霸的抽象化。

### 核心规则框架（基于饮食哲学）

**五味系统**：每道菜有主味属性（咸/鲜/酸/甜/辣）+ 副味。对局时五味之间**相生相克**：
- **相生**：前一道菜"铺垫"了后一道 → 后一道效果增强（先酸开胃，再上重味主菜，主菜效果+30%）
- **相克**：味道冲突 → 互相削弱（极甜后上极辣，两道菜都减分）
- 这意味着**出牌顺序极其重要**——同样的牌组，不同顺序效果完全不同

**上菜节奏**：一局模拟一桌饭——**凉菜→热菜→主菜→汤→主食→甜品**。每个阶段可出的牌类型不同。组牌时要考虑"这桌饭的节奏"。

**荤素搭配**：连续出同类型（连续三道荤/三道素）会被"腻味惩罚"。混搭有加成。

**份量系统**：每道菜有"份量"值，一桌菜的总份量有上限——大菜占位多但效果强，小吃占位少但灵活。

**评审偏好**：每局的"评审"有不同偏好（偏咸鲜/清淡/重口），玩家需要读评审+调整策略。

### 核心循环

1. **收集菜品卡牌**：通过局外探索/任务/奖励获得
2. **组桌（组牌）**：从手头菜品中选出一桌菜，考虑五味搭配、上菜节奏、荤素平衡、份量限制
3. **对局**：按上菜顺序逐张出牌，与对手交替上菜，评审逐道打分
4. **结算**：总分高的赢。获得新菜品/升级材料/声望
5. **进化**：菜品卡牌可"精进"——同一道菜的不同做法版本

### 成长空间

- **卡池扩展**：天津菜+周边菜系=足够的卡池深度
- **菜品升级**：同一道菜有"家常版""老字号版""宗师版"
- **评审系统**：不同评审=不同meta环境
- **赛制**：日赛/周赛/主题赛（"纯素宴""全鱼桌""早点大战"）
- **对战模式**：PvP 排位、PvE 剧情关卡（打各区的名店）
- **套餐系统**：特定组合形成"套餐"——"天津三绝"（狗不理+十八街+耳朵眼）有额外效果

### 风险与待解决

- **数值平衡**是卡牌游戏的永恒难题
- **如何让"菜"不变成纯数字**——视觉呈现每张牌的美术得足够让人馋
- **付费模型**：抽卡 vs 买断的取舍

## 方向三 · 开饭馆（模拟经营）

### 为什么玩

**从无到有的建设快感 + 菜单策略 + 与其他饭店的竞争刺激**。不是"做菜模拟器"，是"饭馆老板模拟器"——核心决策不是怎么切菜，是**上什么菜、怎么定价、客人来了怎么安排、对面新开的火锅店抢走客人怎么办**。

### 玩法目标

把一家小馆子从街边摊做成天津名店。最终目标可以是"入选天津老字号"或"拿到全市评审第一"或"把分店开遍天津九区"。

### 核心循环

1. **获取菜品（局外）**：城市探索、拜访老师傅、发现隐藏食谱、完成任务 → 学到新菜
2. **组菜单**：从已学会菜品中选出当日菜单，考虑：
   - **客群匹配**：附近是工人多还是学生多？早餐店还是晚餐馆？
   - **成本控制**：食材有价格波动，高端菜利润高但出餐慢
   - **菜单搭配**：荤素比例、口味覆盖、招牌菜+引流菜的组合
3. **营业**：客人进店 → 点菜 → 出餐（可简化/自动化）→ 客人评价
4. **竞争事件**：对面新开店/本区美食节/食材断供/美食博主探店 → 玩家需要应对
5. **扩张**：赚钱 → 装修/扩大/招人/开分店/学新菜

### "拼菜"竞争环节

- **美食擂台**：定期举办，你和 AI 饭店（或 PvP）各出菜单，评审打分
- **抢客大战**：同一条街几家店，客人根据菜单、价格、口碑做选择
- **食材竞拍**：稀缺食材在菜市场竞拍
- **拼菜规则**可直接复用 E16 的五味系统——你的菜单 vs 对手的菜单，用同一套搭配/节奏/荤素逻辑评分

### 成长空间

- **饭馆规模**：摊位 → 小馆 → 大店 → 连锁 → 老字号
- **菜品库**：5 道菜起步 → 目标 100+
- **员工系统**：招厨师，不同厨师擅长不同菜系
- **装修/风格**：影响客群定位和口碑
- **区域扩展**：南开区 → 和平区 → 滨海新区 → 全天津
- **声望/评级**：街坊口碑 → 区级名店 → 市级名店 → 老字号
- **历史事件**：经历不同年代的经济环境变化（如果做时间跨度）

### 风险与待解决

- **与现有模拟经营的差异化**：市面上餐厅经营游戏很多（Overcooked、Plate Up、Restaurant Story）。差异化必须来自：
  - **"菜品收集"是核心资产**（而不是解锁菜谱就完事）
  - **天津城市场景**（局外探索有内容）
  - **拼菜对决**（大多数经营游戏没有 PvP/对抗环节）
- **"做菜"过程要不要玩**：
  - Overcooked 式可能太偏操作
  - 完全自动化又少了参与感
  - 中间态：普通出餐自动化，VIP客人/竞赛/新菜首秀时玩家做关键决策（火候/调味微调）
- **局外探索的比重**：太轻则经营变成纯数字游戏；太重则节奏被拖慢

## 三个方向的关系速览

| | A4 胃是资源 | E16 菜品对决 | 开饭馆经营 |
|---|---|---|---|
| **核心动词** | 选/规划 | 组牌/出牌 | 经营/竞争 |
| **独立成游戏？** | 偏薄，适合做子系统 | 能独立 | 能独立 |
| **与其他方向的接口** | 可做经营的"客人端视角" | 可嵌入经营的"拼菜"环节 | 可做主框架 |
| **内容量需求** | 低 | 中 | 中 |
| **玩法深度** | 中 | 高 | 高 |
| **局外探索接口** | B5 一菜一故事 | B5 一菜一故事 | B5 一菜一故事 |

---

## 已确认的备用模块

- **B5 一菜一故事**：作为局外探索层备用。每道菜=一个微叙事+人物，玩家通过城市探索获得菜品的过程承载城市氛围。
- **天津特色**：早点之都/小吃压过大菜/市井气压过精致/菜品组合性强/相声气质。
