# 菜品对决 Demo 开发文档

## 当前状态

单文件 HTML demo，浏览器打开即可玩。验证"共享圆桌 + 口味连击 + 确定性NPC"的核心循环。

**文件**：`demo/dish-duel.html`（HTML + CSS + JS 全部内联）

---

## 核心规则（Demo 当前实现）

- **10格共享圆环**，36°间距，不分半区
- 玩家嘴位固定在 slot 5（底部），NPC嘴位固定在 slot 0（顶部），相隔180°
- 嘴位是固定锚点，菜品随桌面旋转经过嘴位时被吃
- 每回合 **4 AP**，唯一操作：**转桌**（转N格 = N AP）
- 转桌后双方嘴位同时吃菜（双刃剑：你伤NPC的同时自己也吃）
- **饱食度达 50 则输**
- 每轮结束后系统自动补满空格（从菜品池随机抽取）

---

## 口味连击系统

连续吃到有**共同口味标签**的菜，触发连击倍率：

| 连击 | 倍率 | 条件 |
|------|------|------|
| 首次吃 | ×1 | 基线 |
| 2连 | ×2 | 与上一道有共同口味 |
| 3连 | ×3 | 继续有共同口味（上限） |
| 断链 | ×1 | 无共同口味 |

- 双方各自独立追踪combo
- Combo在**NPC回合结束时**重置（不是玩家回合结束时）
- NPC的±1可以接续玩家的自伤combo

---

## NPC AI：确定性±1规则

```
NPC每回合：
  1. 看玩家嘴位左右两格（slot 4 和 slot 6）
  2. 哪个菜的饱食值更高，就往那个方向转1步
  3. 两侧都空则不转
```

- 100%可预测，玩家可以据此规划"出口策略"
- NPC永远以×1倍率吃菜（每轮只吃1次 + combo轮末重置）

## 回合流程

```
玩家回合（4 AP）：
  转桌：拖动圆盘选择方向和步数（1-5步）
  每次转桌消耗 |步数| AP
  可多次转桌（如：+1, +1, +1 = sweep连击 = 3AP）
  点击"结束回合"按钮主动结束
    ↓
NPC回合（自动执行，600ms延迟）：
  确定性±1转桌（看玩家嘴位两侧哪个重）
    ↓
回合结束处理：
  清除位置buff → 补菜填满空格 → 重置双方combo → 回合+1 → 回到玩家回合
```
