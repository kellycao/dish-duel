# 计划：菜品对决 可玩 Demo 开发

## Context

基于 E16 设计文档（当前方案：分区对决），开发一个 HTML+JS 单文件网页 demo，浏览器打开即可玩。目的是验证"双刃剑转桌"核心循环是否有趣。

## 核心规则（Demo 版）

- 圆桌分两个半区，每半区 **4 格**（共 8 格）
- 双方各有一个固定"嘴位"（格子 0）
- NPC 每回合在玩家半区摆 1 道菜
- 玩家每回合 **3 AP**，可做：摆菜(1AP)、换位(1AP)、转桌(1AP)、吃防御菜(1AP)
- 转桌时选择旋转 1-3 格，双方同时吃嘴位上的菜（对称结算/双刃剑）
- 吃到菜 = 饱食度增加该菜的饱食值，菜被消除
- 饱食度达 **15** 则输
- 开局双方半区各预置 2 道菜
- 玩家牌库 12 道菜，每回合抽 1，手牌上限 4

## 开发步骤（8 步）

### Step 1：游戏数据层
- 定义菜品数据结构（名称、饱食值、口味标签、品类标签、效果类型+参数）
- 定义游戏状态（双方饱食度、桌面 8 格状态、手牌、牌库、AP、回合数、当前阶段）
- 定义所有 12+2 道玩家菜 + 5 道 NPC 菜的数据
- 纯 JS 对象，不涉及 UI

### Step 2：核心逻辑 — 转桌结算
- 实现转桌函数：输入旋转格数（1-3），输出双方各吃什么菜、伤害多少
- 实现标签匹配逻辑（同类加成/异类加成）
- 实现位置效果（相邻+1、交换位置、自动再转1格）
- 实现防御菜效果（减饱食度、减半 debuff）
- 实现胜负判定
- 记录"上一次被吃的菜"用于标签加成判定

### Step 3：回合流程
- 实现完整回合循环：NPC摆菜 → 玩家抽牌 → 玩家行动（3AP）→ 回合结束
- 实现 AP 消耗与行动合法性检查
- 实现 NPC AI（随机选菜 + 最近空格放置）
- 实现冷启动（开局预置菜品）
- 实现牌库抽牌 + 手牌管理

### Step 4：UI 布局 — 圆桌可视化
- 用 CSS/Canvas 画圆桌：中间分割线、两个半区各 4 格
- 标注嘴位（嘴图标或高亮）
- 格子里显示菜品卡片（名称 + 饱食值 + 口味标签色块）
- 区分玩家的菜（蓝色系）和 NPC 的菜（红色系）
- 显示双方饱食度条、当前 AP

### Step 5：UI 交互 — 玩家操作
- 摆菜：点击手牌 → 点击对方半区空格 → 消耗 1AP
- 转桌：点击"转 1/2/3 格"按钮 → 播放旋转动画 → 结算 → 显示吃了什么菜、各受多少伤
- 换位：点击对方半区己方菜 → 点击另一个格子 → 交换
- 吃防御菜：点击手牌中的防御菜 → 立即生效
- 结束回合按钮（AP 用完时自动结束）

### Step 6：结算动画与反馈
- 转桌时的旋转动画（CSS transition 或简单角度变化）
- 吃到菜时的高亮闪烁 + 伤害数字飘字
- 标签加成/位置效果触发时的额外提示（如"+2 同类加成！"）
- 回合开始/结束的过渡提示
- 胜负画面

### Step 7：信息面板与辅助
- 鼠标悬停菜品显示详细信息（效果描述、标签）
- 转桌前预览：hover 转桌按钮时高亮"双方嘴位将对应哪道菜"
- 战斗日志（文字列表，记录每次操作和结算）
- 当前回合数和阶段指示

### Step 8：打磨与可玩性
- NPC 摆菜时的简单动画
- 音效（可选，用 Web Audio API 做简单音效）
- 开局说明弹窗（规则提示）
- 重新开始按钮
- 整体视觉风格调整（食物主题配色）
